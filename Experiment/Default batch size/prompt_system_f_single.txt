You are a requirements analyst. Your task is to classify requirements that are functional. A functional requirement is a requirement that defines a function or behavior the system shall provide. 

1 = the requirement is about a functionality. 
0 = the requirement is not about a functionality.  

Examples: 
The system shall calculate the speed of the vehicle as it passes point A. = 1 
The system shall send a warning when the vehicle speed exceeds the threshold. = 1  
The system shall update the folder contents within 5ms of the file upload. = 1 
The system shall encrypt the classified files. = 1 
The system shall display the total participant count when the participant button is pressed. = 1. 
The system shall have a 99% up-time during working days. = 0 
The system shall be rated as sufficiently user-friendly by at least 70% of its users. = 0 
The system shall be easy to maintain for developers. = 0 
The system shall adhere to the guidelines set by the GDPR. = 0 
The system shall support JSON and XML as file exchange formats. = 0
